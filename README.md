# CYBER ZONE — WebSocket архитектура (v5)

Гибридная реализация по образцу Hexagonal / Clean Architecture:

| Слой | Где | Что |
|------|-----|-----|
| **L1 ДАННЫЕ** | `club/core.py` (Python) + JSON (JS) | `World`, `Session`, `Client` как `namedtuple` → сериализуются в JSON |
| **L2 ЛОГИКА** | `club/core.py` (только Python) | `tick`, `apply_command` — 100% чистые функции |
| **L3 ОБОЛОЧКА** | `club/server.py` (FastAPI WebSocket) + `web/index.html` (JS+SVG) | Все побочные эффекты: сеть, цикл, отрисовка, кнопки |

## Структура

```
club_websocket/
├── club/
│   ├── __init__.py
│   ├── core.py        ← L1+L2: чистая логика, ноль зависимостей кроме stdlib
│   └── server.py      ← Python L3: WebSocket-сервер
└── web/
    └── index.html     ← JS L3: одностраничный фронтенд (HTML+CSS+JS)
```

## Запуск

```bash
pip install fastapi uvicorn
python -m club_websocket.server
```

Откройте http://127.0.0.1:8000/ в браузере.

## Команды

Кнопки в браузере шлют интенты на сервер; сервер обновляет состояние через
чистые функции `core.apply_command()` и рассылает новый снапшот всем
подключённым клиентам.

| Кнопка | Сообщение |
|--------|-----------|
| СТАРТ | `{"action": "start"}` |
| ПАУЗА | `{"action": "pause"}` (toggle) |
| СБРОС | `{"action": "reset"}` |
| НОВЫЙ ДЕНЬ | `{"action": "new_day"}` |
| 1×/2×/5×/10×/20×/60× | `{"action": "speed", "value": N}` |

## Опциональные файлы

В рабочей директории (или в `club/`) можно положить:

- `names.txt`, `surnames.txt` — формат: `Имя м` или `Фамилия ж` (по одной записи на строку).
- `chat_questions.txt`, `chat_answers.txt`, `chat_emotions.txt`, `chat_reactions.txt`,
  `live_questions.txt`, `live_answers.txt`, `live_reactions.txt` — фразы для чатов
  (по одной на строку).

Если файлов нет — используется встроенный fallback.

## Архитектурные особенности

1. **Чистота L2.** `core.tick(state) → state` не делает ни одного I/O. Это позволяет
   тестировать симуляцию детерминированно (один и тот же seed даёт один и тот же ход).
2. **Интенты вместо мутаций.** Фронтенд не меняет состояние сам — он шлёт интент на
   сервер, тот применяет чистую функцию-редьюсер, и обновлённое состояние
   распространяется обратно. Это сохраняет single source of truth.
3. **Ограничение трафика.** `world_to_dict()` обрезает `logs`/`chats` до последних
   80 записей и не отправляет тяжёлые поля (`rng`, словари имён и фраз). Снапшот
   с 4 активными сессиями весит ~6 KB.
4. **Авто-переподключение.** Если сервер перезапустился, фронтенд переподключается
   через 1.5 секунды.
